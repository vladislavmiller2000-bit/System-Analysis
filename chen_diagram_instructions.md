# Диаграмма Чена для сущности ЗАКАЗ
## Инструкция для создания в diagrams.net

### Основные элементы для создания:

#### 1. Основная сущность (Entity - прямоугольник):
```
┌─────────────────┐
│     ЗАКАЗ       │
└─────────────────┘
```

---

### 2. ПЕРВИЧНЫЙ КЛЮЧ (Primary Key - эллипс с подчеркиванием):

| Атрибут | Тип | Описание |
|---------|-----|---------|
| **№ заказа** | PK | Уникальный идентификатор заказа (подчеркнуть) |

---

### 3. ПРОСТЫЕ АТРИБУТЫ (Simple Attributes - овалы):

| # | Атрибут | Описание |
|---|---------|---------|
| 1 | Дата заказа | Дата оформления заказа |
| 2 | Время заказа | Время оформления заказа |
| 3 | Статус заказа | Статус: оформлен, оплачен, доставлен, отменён |
| 4 | Способ оплаты | Вид оплаты (без сохранения данных карт) |
| 5 | ID Пользователя | Внешний ключ (FK) - связь с пользователем |

---

### 4. СОСТАВНЫЕ АТРИБУТЫ (Composite Attributes - овал с линиями к компонентам):

**A. Адрес доставки** (состоит из):
- Страна
- Город
- Улица
- Номер дома
- Квартира
- Почтовый индекс

**B. Сведения о доставке** (состоит из):
- Способ доставки (курьер/самовывоз/почта)
- Дата доставки

---

### 5. МНОЖЕСТВЕННЫЕ АТРИБУТЫ (Multi-valued Attributes - двойной овал):

**Товары в заказе** (каждый товар содержит):
- Артикул товара
- Название товара
- Количество
- Цена за единицу
- Размер скидки (если применима акция)

---

### 6. ПРОИЗВОДНЫЕ АТРИБУТЫ (Derived Attributes - пунктирный овал):

| # | Атрибут | Формула вычисления |
|---|---------|-------------------|
| 1 | Сумма заказа | = Σ(Цена за единицу × Количество × (1 - Скидка)) |
| 2 | Сумма скидок | = Σ(Размер скидки для каждого товара) |

---

## Инструкции для создания в diagrams.net:

### Шаг 1: Создание основной сущности
1. Вставьте **прямоугольник** в центр диаграммы
2. Напишите "ЗАКАЗ" в центре
3. Измените цвет на желтоватый (#FFF9E6)

### Шаг 2: Добавление первичного ключа
1. Создайте **эллипс** над сущностью
2. Напишите **подчеркнутый** текст "№ заказа"
3. Добавьте пометку "(PK)"
4. Соедините **полной линией** с сущностью

### Шаг 3: Добавление простых атрибутов
1. Создайте **эллипсы** вокруг сущности
2. Напишите названия атрибутов
3. Соедините **полными линиями** с сущностью
4. Расположите по часовой стрелке

### Шаг 4: Добавление составных атрибутов
1. Создайте **эллипсы** для "Адрес доставки" и "Сведения о доставке"
2. Создайте **эллипсы для каждого компонента**
3. Соедините компоненты с родительским атрибутом **полными линиями**
4. Соедините составной атрибут с сущностью **полной линией**

### Шаг 5: Добавление множественного атрибута
1. Создайте **двойной овал** (нарисуйте два овала)
2. Напишите "Товары в заказе"
3. Создайте **эллипсы для компонентов**: Артикул, Название, Количество, Цена
4. Соедините компоненты с множественным атрибутом **полными линиями**
5. Соедините множественный атрибут с сущностью **утолщённой линией** (stroke-width: 3)

### Шаг 6: Добавление производных атрибутов
1. Создайте **пунктирные эллипсы** для "Сумма заказа" и "Сумма скидок"
2. Напишите названия атрибутов
3. Добавьте пометку "(вычисляемый)"
4. Соедините **пунктирными линиями** с сущностью

### Шаг 7: Добавление легенды
Добавьте в нижней части диаграммы таблицу с обозначениями:
```
─── Простой атрибут
─ ─ ─ Производный атрибут
═══ Множественный атрибут
PK = Первичный ключ
FK = Внешний ключ
```

## Допущения:

1. **Дата и время заказа** - можно объединить в один составной атрибут "Дата-время заказа" или оставить отдельными
2. **Адрес доставки** - может быть ссылкой на отдельную сущность Адрес (если расширять модель)
3. **Товары в заказе** - множественный атрибут содержит информацию о каждом товаре в заказе с количеством и ценой
4. **Скидки** - применяются на уровне товара (не на весь заказ), поэтому размер скидки входит в Товары в заказе
5. **ID Пользователя** - внешний ключ, указывающий на сущность Пользователь (не отображаем саму сущность по заданию)

---

## Примечание:

Эта диаграмма показывает только сущность **ЗАКАЗ** согласно требованиям задания. 
При необходимости расширения модели можно добавить дополнительные сущности:
- Пользователь
- Товар
- Акция
- Категория товара

Но в текущем задании они не требуются.
