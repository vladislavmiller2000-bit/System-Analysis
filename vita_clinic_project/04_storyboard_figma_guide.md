# Раскадровка (Storyboard) — Руководство для работы в Figma

## 1. Анализ: Функции приложения (из DFD)

Диаграмма потоков данных определяет **3 основных процесса**:

| # | Процесс | Описание | Хранилища данных |
|---|---------|----------|-----------------|
| 1 | **Проверить записи на приём** | Авторизация по телефону + OTP, просмотр личного кабинета и записей | D1 Пользователи, D3 Записи, Система OTP |
| 2 | **Найти врача** | Просмотр списка врачей, поиск/фильтрация, просмотр карточки врача | D2 Врачи |
| 3 | **Записаться на приём** | Ввод данных о приёме, получение статуса записи | Система онлайн-записи |

**Внешняя сущность:** Пользователь (пациент)

## 3. Пользовательские потоки (переходы между экранами)

### Поток 1: Главная → Мои записи (проверка записей)
```
0 Главная → [таб "Мои записи"] → 4 Ввод телефона → 5 Ввод OTP → 6 Данные пользователя → 7 Список записей
```

### Поток 2: Главная → Врачи (поиск врача)
```
0 Главная → [таб "Врачи"] → 8 Список врачей → 9 Карточка врача
                                    ↓
                              10 Поиск/фильтр → 8 Список врачей (отфильтрованный)
```

### Поток 3: Главная → Записаться на приём
```
0 Главная → [таб "Записаться"] → 11 Ввод данных о приёме → 12 Статус записи
```

**Альтернативный вход:** Из карточки врача (экран 9) → 11 Ввод данных о приёме

## 4. Детальное описание каждого вайрфрейма

### Экран 0: Главная (уже готов)
- Логотип клиники
- Лента новостей (фото + заголовок + описание)
- Нижняя навигация: Главная | Записаться | Врачи | Мои записи

---

### Экран 4: Ввод данных пользователя (уже готов)
- Заголовок «Авторизация»
- Поле ввода телефона: +7 (xxx) xxx xx xx
- Кнопка «Получить код» / «Войти»
- Нижняя навигация (таб «Мои записи» активен)

---

### Экран 8: Просмотр списка врачей (НУЖНО НАПОЛНИТЬ)

**Данные из DFD:** Процесс 2 получает данные из хранилища D2 (Врачи). Пользователь отправляет «Запрос списка врачей» и получает «Список врачей».

**Из ограничений:** Каждый врач привязан к филиалу. Связка ФИО + специальность уникальна.

**Элементы интерфейса:**
- Заголовок «Наши врачи» (уже есть)
- Иконка поиска 🔍 (из шаблонов — символ «Символы»)
- Иконка фильтра ⚙ (из шаблонов)
- Список карточек врачей, каждая содержит:
  - **Photo** — фото врача (заглушка из шаблонов)
  - **Текстовый блок (Заголовок)** — ФИО врача (например, «Иванов И.И.»)
  - **Текстовый блок (обычный текст)** — Специальность (например, «Терапевт»)
  - **Текстовый блок (обычный текст)** — Филиал (например, «Клиника на Ленина, 10»)
- Нижняя навигация (таб «Врачи» активен)

---

### Экран 11: Ввод данных о приёме (НУЖНО ДОПОЛНИТЬ)

**Данные из DFD:** Процесс 3 — пользователь отправляет «Запрос на запись на приём» в систему онлайн-записи.

**Уже есть:** Заголовок «Записаться на приём», выпадающий список «Врач» (Иванов И.И.)

**Нужно добавить:**
- Выпадающий список **«Специальность»** (или привязать автоматически к врачу)
- Выпадающий список **«Филиал»** (автоподстановка по врачу, т.к. врач привязан к филиалу)
- Поле ввода или выпадающий список **«Дата приёма»** (формат: ДД.ММ.ГГГГ)
- Выпадающий список **«Время приёма»** (доступные слоты)
- Поле ввода **«Телефон»** (для связи / подтверждения)
- Поле ввода **«ФИО пациента»** (опционально, можно подтянуть из авторизации)
- **Кнопка «Записаться»**
- Нижняя навигация (таб «Записаться» активен)

---

### Экран 5: Ввод OTP (НУЖНО СОЗДАТЬ)

**Данные из DFD:** Система OTP отправляет OTP-код пользователю. Пользователь вводит «OTP для валидации», система возвращает «Статус валидации OTP».

**Элементы интерфейса:**
- Стрелка «Назад» ← (из шаблонов)
- Заголовок «Введите код»
- Текстовый блок (обычный): «Мы отправили SMS с кодом на номер +7 (xxx) xxx xx xx»
- Поле ввода: XXX-XXX (код OTP)
- Кнопка «Подтвердить»
- Текстовый блок (обычный): «Отправить код повторно» (ссылка)
- Нижняя навигация (таб «Мои записи» активен)

---

### Экран 5а: Ошибка OTP (НУЖНО СОЗДАТЬ — экран ошибки)

**Принцип UI:** Терпимость к ошибкам — пользователь должен понимать, что произошло, и иметь возможность исправить ситуацию.

**Элементы интерфейса:**
- Стрелка «Назад» ← (из шаблонов)
- Заголовок «Введите код»
- Текстовый блок (обычный): «Мы отправили SMS с кодом на номер +7 (xxx) xxx xx xx»
- 4 поля OTP с **красной рамкой** (введённый код ошибочен)
- **Текст ошибки (красный):** «Неверный код. Проверьте SMS и попробуйте ещё раз»
- Кнопка «Попробовать снова»
- Текстовый блок (обычный): «Отправить код повторно» (ссылка)
- Нижняя навигация (таб «Мои записи» активен)

---

### Экран 6: Просмотр данных пользователя (НУЖНО СОЗДАТЬ)

**Данные из DFD:** Из D1 (Пользователи) процесс 1 получает данные пользователя.

**Элементы интерфейса:**
- Стрелка «Назад» ←
- Иконка профиля (аватар из шаблонов)
- Заголовок «Личный кабинет»
- Текстовый блок (Заголовок): ФИО пользователя
- Текстовый блок (обычный): Номер телефона
- Кнопка «Мои записи» (переход к экрану 7)
- Кнопка «Выйти» (опционально)
- Нижняя навигация (таб «Мои записи» активен)

---

### Экран 7: Просмотр записей на приём (НУЖНО СОЗДАТЬ)

**Данные из DFD:** Из D3 (Записи) процесс 1 возвращает пользователю «Данные о записи».

**Элементы интерфейса:**
- Стрелка «Назад» ←
- Заголовок «Мои записи»
- Список карточек записей, каждая содержит:
  - **Текстовый блок (Заголовок)** — ФИО врача + Специальность
  - **Текстовый блок (обычный)** — Дата и время приёма
  - **Текстовый блок (обычный)** — Филиал / Адрес клиники
  - **Текстовый блок (обычный)** — Статус записи (Подтверждена / Ожидает)
- Нижняя навигация (таб «Мои записи» активен)

---

### Экран 9: Просмотр данных врача (НУЖНО СОЗДАТЬ)

**Данные из DFD:** Пользователь отправляет «Запрос данных врача» → получает «Данные о враче» из D2.

**Из ограничений:** Врач привязан к конкретному филиалу.

**Элементы интерфейса:**
- Стрелка «Назад» ←
- **Photo** — фото врача (крупная заглушка)
- **Текстовый блок (Заголовок)** — ФИО врача
- **Текстовый блок (обычный)** — Специальность
- **Текстовый блок (обычный)** — Филиал / Адрес клиники
- **Текстовый блок (обычный)** — Описание / Стаж / Образование (опционально)
- **Кнопка «Записаться к врачу»** → переход на экран 11 с предвыбранным врачом
- Нижняя навигация (таб «Врачи» активен)

---

### Экран 10: Поиск / Фильтрация врача (НУЖНО СОЗДАТЬ)

**Данные из DFD:** Пользователь формирует «Запрос списка врачей» с параметрами.

**Элементы интерфейса:**
- Стрелка «Назад» ←
- Заголовок «Найти врача»
- Поле ввода: **Поиск по ФИО** (XXX-XXX-XXX-XXX)
- Выпадающий список: **Специальность** (Например: Терапевт, Хирург...)
- Выпадающий список: **Филиал** (Например: Клиника на Ленина...)
- Кнопка «Найти» → переход на экран 8 с отфильтрованным списком
- Нижняя навигация (таб «Врачи» активен)

---

### Экран 12: Статус записи на приём (НУЖНО СОЗДАТЬ)

**Данные из DFD:** Система онлайн-записи возвращает пользователю «Статус записи на приём».

**Элементы интерфейса:**
- Заголовок «Запись оформлена» (или «Статус записи»)
- Текстовый блок (обычный): Статус — «Подтверждена» / «Ожидает подтверждения»
- Сводка записи:
  - **Текстовый блок (Заголовок)** — ФИО врача
  - **Текстовый блок (обычный)** — Специальность
  - **Текстовый блок (обычный)** — Дата и время
  - **Текстовый блок (обычный)** — Филиал / Адрес
- Кнопка «На главную» → переход на экран 0
- Кнопка «Мои записи» → переход на экран 4 (авторизация) → 7
- Нижняя навигация (таб «Записаться» активен)

---

### Экран 12а: Ошибка записи на приём (НУЖНО СОЗДАТЬ — экран ошибки)

**Принцип UI:** Терпимость к ошибкам — при неуспешной попытке записи пользователь видит понятное объяснение и может повторить действие.

**Элементы интерфейса:**
- Иконка ошибки ✕ (красный круг)
- Заголовок «Не удалось создать запись»
- Бейдж «Ошибка» (красный)
- Карточка с текстом ошибки: «Выбранное время уже занято или произошла ошибка сервера. Пожалуйста, попробуйте выбрать другое время.»
- Кнопка «Попробовать снова» → переход на экран 11
- Кнопка «На главную» → переход на экран 0
- Нижняя навигация (таб «Записаться» активен)

## 7. Соответствие принципов UI

| Принцип | Как применить |
|---------|--------------|
| **KISS** | Минимум элементов на экране, одно действие — один экран |
| **Предсказуемость** | Единая нижняя навигация на всех экранах, стандартные паттерны (← назад, поля, кнопки) |
| **Терпимость к ошибкам** | Кнопка «Назад» на каждом экране, повторная отправка OTP, экран ошибки OTP (5а), экран ошибки записи (12а) |
| **Закон Фиттса** | Кнопки действий (CTA) крупные, расположены удобно для большого пальца |
| **Группировка** | Данные врача сгруппированы в карточку; поля формы записи логически упорядочены |
| **Поддержка контекста** | Активный таб в навигации показывает текущий раздел; заголовки экранов |
| **Эстетика** | Консистентные отступы, единый стиль элементов из шаблонов |
