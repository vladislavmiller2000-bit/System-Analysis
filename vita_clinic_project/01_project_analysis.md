# Проект: Автоматизация записи на приём в медицинской клинике "Вита"

## 📋 Общая информация о проекте

**Заказчик:** Сеть медицинских клиник "Вита" имени Е.Л. Яницкой  
**Исполнитель:** RANA.Software  
**Цель проекта:** Разработка приложения для пациентов с функцией самостоятельной записи на приём к врачу

### Текущая ситуация (AS-IS)
- Запись на приём осуществляется только по телефону через администратора
- Каждый филиал имеет свою систему в виде электронного календаря
- Доступ к календарю имеют врач и администратор конкретного филиала
- Средний возраст пациентов ~ 30 лет (целевая аудитория для мобильного приложения)

### Целевое состояние (TO-BE)
- Пациенты могут самостоятельно записываться через приложение
- Автоматизация процесса подтверждения записи
- Устранение роли администратора в процессе записи

## 👥 Участники процесса (Акторы)

### AS-IS (текущий процесс):
1. **Пациент** - инициирует запись, приходит на приём
2. **Администратор** - принимает звонки, управляет календарём, подтверждает записи
3. **Врач** - проводит приём, ставит диагноз, назначает лечение

### TO-BE (будущий процесс):
1. **Пациент** - самостоятельно записывается через приложение, подтверждает запись
2. **Врач** - проводит приём (без изменений)
3. **Система/Приложение** - заменяет функции администратора по записи

## 📊 Сравнение AS-IS и TO-BE

| Аспект | AS-IS | TO-BE |
|--------|-------|-------|
| Способ записи | По телефону | Через приложение |
| Поиск врача | Администратор помогает | Пациент сам выбирает |
| Выбор времени | Администратор предлагает | Пациент сам видит слоты |
| Подтверждение | Звонок администратора | Уведомление в приложении |
| Отмена записи | Через администратора | Самостоятельно в приложении |
| Роль администратора | Ключевая | Отсутствует в процессе записи |
| Доступность | Рабочие часы | 24/7 |
