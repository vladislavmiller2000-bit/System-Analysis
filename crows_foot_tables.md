# Таблицы Crow's Foot: Полная спецификация

## ТАБЛИЦА 1: ЗАКАЗ

### Структура и колонки

| № | Название колонки | Тип данных | Ключ | NULL? | Описание |
|---|------------------|-----------|------|-------|----------|
| 1 | № заказа | INT | PK | NO | Уникальный номер заказа (автоинкремент) |
| 2 | Дата заказа | DATE | - | NO | Дата оформления заказа |
| 3 | Время заказа | TIME | - | NO | Время оформления заказа |
| 4 | Статус заказа | VARCHAR(20) | - | NO | Статус: 'оформлен', 'оплачен', 'доставлен', 'отменён' |
| 5 | Способ оплаты | VARCHAR(50) | - | NO | Способ оплаты: 'карта', 'счёт', 'наличные' (без сохр. данных карт) |
| 6 | Сумма заказа | DECIMAL(10,2) | - | YES | Общая сумма заказа (производная, вычисляется) |
| 7 | Сумма скидок | DECIMAL(10,2) | - | YES | Общая сумма скидок (производная, вычисляется) |
| 8 | ID Пользователя | INT | FK | NO | Ссылка на таблицу ПОЛЬЗОВАТЕЛЬ |

### Первичный ключ
```
PRIMARY KEY (№ заказа)
```

### Внешние ключи
```
FOREIGN KEY (ID Пользователя) REFERENCES ПОЛЬЗОВАТЕЛЬ(ID)
```

### Индексы (рекомендуется)
```
CREATE INDEX idx_status ON ЗАКАЗ(Статус заказа);
CREATE INDEX idx_date ON ЗАКАЗ(Дата заказа);
CREATE INDEX idx_user ON ЗАКАЗ(ID Пользователя);
```

### SQL для создания таблицы
```sql
CREATE TABLE ЗАКАЗ (
    № заказа INT PRIMARY KEY AUTO_INCREMENT,
    Дата заказа DATE NOT NULL,
    Время заказа TIME NOT NULL,
    Статус заказа VARCHAR(20) NOT NULL CHECK (Статус заказа IN ('оформлен', 'оплачен', 'доставлен', 'отменён')),
    Способ оплаты VARCHAR(50) NOT NULL CHECK (Способ оплаты IN ('карта', 'счёт', 'наличные')),
    Сумма заказа DECIMAL(10,2),
    Сумма скидок DECIMAL(10,2),
    ID Пользователя INT NOT NULL,
    FOREIGN KEY (ID Пользователя) REFERENCES ПОЛЬЗОВАТЕЛЬ(ID),
    CONSTRAINT ck_dates CHECK (Дата заказа <= CURDATE())
);
```

### Примеры данных
```
№ заказа | Дата заказа | Время заказа | Статус заказа | Способ оплаты | Сумма заказа | Сумма скидок | ID Пользователя
---------|-------------|--------------|----------------|---------------|--------------|--------------|------------------
1        | 2024-12-20  | 14:30:00     | доставлен      | карта         | 5500.00      | 500.00       | 101
2        | 2024-12-21  | 10:15:00     | оплачен        | счёт          | 3200.00      | 200.00       | 102
3        | 2024-12-22  | 16:45:00     | оформлен       | наличные      | 1500.00      | 0.00         | 101
```

## ТАБЛИЦА 2: ТОВАР В ЗАКАЗЕ

### Структура и колонки

| № | Название колонки | Тип данных | Ключ | NULL? | Описание |
|---|------------------|-----------|------|-------|----------|
| 1 | № заказа | INT | PK, FK | NO | Номер заказа (ссылка на ЗАКАЗ) |
| 2 | Артикул товара | VARCHAR(50) | PK | NO | Артикул товара (ссылка на ТОВАР) |
| 3 | Количество | INT | - | NO | Количество единиц товара в заказе |
| 4 | Цена за единицу | DECIMAL(10,2) | - | NO | Цена товара на момент заказа |
| 5 | Размер скидки | DECIMAL(5,2) | - | YES | Процент скидки (из акции) |

### Первичный ключ (составной)
```
PRIMARY KEY (№ заказа, Артикул товара)
```
**Обоснование**: 
- Один заказ + один артикул = уникальная позиция в заказе
- Одна позиция не может повторяться в одном заказе

### Внешние ключи
```
FOREIGN KEY (№ заказа) REFERENCES ЗАКАЗ(№ заказа) ON DELETE CASCADE
FOREIGN KEY (Артикул товара) REFERENCES ТОВАР(Артикул) ON DELETE RESTRICT
```

### SQL для создания таблицы
```sql
CREATE TABLE ТОВАР_В_ЗАКАЗЕ (
    № заказа INT NOT NULL,
    Артикул товара VARCHAR(50) NOT NULL,
    Количество INT NOT NULL CHECK (Количество > 0),
    Цена за единицу DECIMAL(10,2) NOT NULL CHECK (Цена за единицу > 0),
    Размер скидки DECIMAL(5,2) DEFAULT 0 CHECK (Размер скидки >= 0 AND Размер скидки <= 100),
    PRIMARY KEY (№ заказа, Артикул товара),
    FOREIGN KEY (№ заказа) REFERENCES ЗАКАЗ(№ заказа) ON DELETE CASCADE,
    FOREIGN KEY (Артикул товара) REFERENCES ТОВАР(Артикул) ON DELETE RESTRICT
);
```

### Индексы (рекомендуется)
```
CREATE INDEX idx_artikul ON ТОВАР_В_ЗАКАЗЕ(Артикул товара);
```

### Примеры данных
```
№ заказа | Артикул товара | Количество | Цена за единицу | Размер скидки
---------|----------------|------------|-----------------|---------------
1        | RUB001         | 2          | 2500.00          | 10.00
1        | RUB002         | 1          | 500.00           | 0.00
2        | RUB003         | 3          | 1000.00          | 5.00
2        | RUB001         | 1          | 2500.00          | 10.00
3        | RUB002         | 1          | 500.00           | NULL
```

### Вычисляемые поля (в приложении)
```
Сумма товара = Количество × Цена за единицу × (1 - Размер скидки / 100)

Пример для строки 1:
2 × 2500.00 × (1 - 10/100) = 2 × 2500 × 0.9 = 4500.00
```

## ТАБЛИЦА 3: СВЕДЕНИЯ О ДОСТАВКЕ

### Структура и колонки

| № | Название колонки | Тип данных | Ключ | NULL? | Описание |
|---|------------------|-----------|------|-------|----------|
| 1 | № заказа | INT | PK, FK | NO | Номер заказа (ссылка на ЗАКАЗ) |
| 2 | Способ доставки | VARCHAR(30) | - | NO | Способ: 'курьер', 'самовывоз', 'почта' |
| 3 | Дата доставки | DATE | - | YES | Ожидаемая или фактическая дата доставки |
| 4 | Страна | VARCHAR(100) | - | NO | Страна доставки |
| 5 | Город | VARCHAR(100) | - | NO | Город доставки |
| 6 | Улица | VARCHAR(100) | - | NO | Улица доставки |
| 7 | Номер дома | VARCHAR(20) | - | NO | Номер дома |
| 8 | Квартира | VARCHAR(20) | - | YES | Номер квартиры (для многоквартирных) |
| 9 | Почтовый индекс | VARCHAR(20) | - | YES | Почтовый индекс |

### Первичный ключ
```
PRIMARY KEY (№ заказа)
```
**Обоснование**: 
- Каждый заказ имеет ровно одно сведение о доставке (1:1)
- Один заказ = одна доставка

### Внешние ключи
```
FOREIGN KEY (№ заказа) REFERENCES ЗАКАЗ(№ заказа) ON DELETE CASCADE
```

### SQL для создания таблицы
```sql
CREATE TABLE СВЕДЕНИЯ_О_ДОСТАВКЕ (
    № заказа INT PRIMARY KEY,
    Способ доставки VARCHAR(30) NOT NULL CHECK (Способ доставки IN ('курьер', 'самовывоз', 'почта')),
    Дата доставки DATE,
    Страна VARCHAR(100) NOT NULL,
    Город VARCHAR(100) NOT NULL,
    Улица VARCHAR(100) NOT NULL,
    Номер дома VARCHAR(20) NOT NULL,
    Квартира VARCHAR(20),
    Почтовый индекс VARCHAR(20),
    FOREIGN KEY (№ заказа) REFERENCES ЗАКАЗ(№ заказа) ON DELETE CASCADE
);
```

### Примеры данных
```
№ заказа | Способ доставки | Дата доставки | Страна | Город     | Улица          | Номер дома | Квартира | Почтовый индекс
---------|-----------------|---------------|--------|-----------|----------------|------------|----------|------------------
1        | курьер          | 2024-12-23    | Россия | Москва    | Ленинский пр.  | 45         | 120      | 119991
2        | почта           | 2024-12-25    | Россия | Санкт-Петербург | Невский пр. | 28 | NULL | 190000
3        | самовывоз       | NULL          | Россия | Москва    | Красная площадь| 1         | NULL     | 109012
```

## ВАЛИДАЦИОННЫЕ ПРАВИЛА

### Для таблицы ЗАКАЗ:
- Дата заказа <= текущей даты
- Статус заказа только из допустимых значений
- Способ оплаты только из допустимых значений
- Сумма заказа >= 0
- Сумма скидок >= 0

### Для таблицы ТОВАР В ЗАКАЗЕ:
- Количество > 0
- Цена за единицу > 0
- Размер скидки между 0 и 100 (включительно)
- При удалении заказа удаляются все его товары (ON DELETE CASCADE)

### Для таблицы СВЕДЕНИЯ О ДОСТАВКЕ:
- Способ доставки только из допустимых значений
- Все поля адреса заполнены (не NULL)
- При удалении заказа удаляются его сведения о доставке (ON DELETE CASCADE)

## ОТНОШЕНИЯ МЕЖДУ ТАБЛИЦАМИ

```
ПОЛЬЗОВАТЕЛЬ
    ↓ (1:N, неидентифицирующая)
ЗАКАЗ
    ├─ (1:N, идентифицирующая) → ТОВАР_В_ЗАКАЗЕ
    └─ (1:1, идентифицирующая) → СВЕДЕНИЯ_О_ДОСТАВКЕ
```

## МОЩНОСТИ И КАРДИНАЛЬНОСТЬ

| Связь | Кардинальность | Тип | Описание |
|-------|----------------|-----|---------|
| ПОЛЬЗОВАТЕЛЬ → ЗАКАЗ | 1:N | Неидентифицирующая | Пользователь может оформить много заказов |
| ЗАКАЗ → ТОВАР В ЗАКАЗЕ | 1:N | Идентифицирующая | Заказ содержит много товаров |
| ЗАКАЗ → СВЕДЕНИЯ О ДОСТАВКЕ | 1:1 | Идентифицирующая | Заказ имеет одно сведение о доставке |
